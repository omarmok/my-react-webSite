Start Developing

PS-Design pages load bootstrap-v5.css alongside a custom main.css that compiles from the SCSS source files.

Overview

We rely on Bootstrap 5 for the base UI scaffolding, then customize the brand, colors, and typography via SCSS and compile everything into main.css.

  • Load bootstrap-v5.css first, then main.css for overrides.
  • Design updates should happen in SCSS, not by editing main.css directly.

Stylesheets

These are the core stylesheets loaded on PS-Design pages.

File Purpose
./css/bootstrap-v5.css Bootstrap 5 base framework for layout and components.
./css/main.css Custom UI stylesheet that bundles design tokens and overrides.

SCSS Sources

main.css is compiled from these SCSS sources under the css folder.

File Role
css/helpers/_index.scss Primary aggregator that imports helper and token files into main.css.
css/helpers/_variables.scss Defines color, spacing, and radius tokens that cascade into every component.
css/helpers/_mixins.scss Typography mixins, generated .font-* utilities, and shared component helpers.
css/_customClasses.scss Custom utility classes (such as .font-*, .gap-*, .radius-*) used directly in HTML.
css/helpers/_changeFont.scss Defines IBM Plex Sans Arabic (multiple weights) used when .cairo is applied.
css/helpers/_changeFontClasses.scss Applies Arabic font styles across UI elements and adjusts default text sizes.
css/helpers/_fonts.scss Defines 29LT Bukra font-face files located in the fonts folder.

Workflow

  • Update SCSS files under css and rebuild main.css.
  • Add new tokens in _variables.scss and consume them in components.
  • Keep Bootstrap 5 conventions by layering on top of its base classes.
