Iconography

The PS Design iconography system relies on a shared registry, a loader that clones glyphs at runtime, and a single set of SCSS helpers that preserve performance, consistency, and accessibility.

Overview

PS Design now documents one icon system that stretches from registry through styling.

Registry

All glyphs live inside icons.svg as inline <svg> tags that keep their original viewBox and drop width/height and hardcoded colors. Each entry carries a descriptive, kebab-case data-icon that is globally unique.

Loader

icon.js fetches the registry once, finds the hidden #svg-registry, and registers every glyph in a map. When the DOM contains [data-icon], the loader clones the matching template (cloneNode(true)) and injects it directly; no <use> references or manual imports are needed.

Styling

icon.scss (compiled into main.css) sets the base .icon, default size (1.25rem), sizing helpers such as .icon-2xl, semantic colors like .Primary-600, and spacing helpers that keep glyphs aligned.

Usage rules

  • Use data-icon on empty hosts; avoid pasting inline SVGs.
  • Assign unique registry names in icons.svg; do not duplicate identifiers.
  • Combine size and color helpers (e.g., .icon-2xl + .Primary-600) instead of tweaking the host.
  • Decorative icons get aria-hidden="true"; provide the text label elsewhere.
  • Keep icons.svg and icon.js in sync so the loader can pull new glyphs.

Adding a new icon

Follow these steps before referencing a new glyph in the UI.

  1. Add an inline <svg> to icons.svg under the hidden registry container.
  2. Keep the original viewBox, remove any width/height, and strip fill or stroke attributes so currentColor can control them.
  3. Give the glyph a unique, descriptive kebab-case value in data-icon and keep that name stable.
  4. Do not place this SVG directly in other pages; rely on the loader to clone it wherever it is needed.
<div id="svg-registry" aria-hidden="true" style="display:none">
  <svg data-icon="weather-storm" viewBox="0 0 24 24" xmlns="http://www.w3.org/2000/svg">
    <path d="M6 12h12v6a3 3 0 0 1-3 3H9a3 3 0 0 1-3-3z"/>
  </svg>
</div>

Anatomy

Every component copies the same empty markup; icon.js fills it in at runtime.

Canonical markup

<span class="icon icon-2xl Primary-600" data-icon="car"></span>

The blank <span> is a lightweight placeholder that icon.js replaces with the cloned SVG.

Host element

<span> keeps the markup inline-friendly, preserves line height, and lets the loader remove the placeholder before appending the glyph.

.icon base class

The CSS sets inline-flex centering, a 1.25rem default size, and color: currentColor so the glyph inherits tint tokens.

Size helper

.icon-2xl (and its siblings) override the width/height to fixed rem values; the loader keeps the SVG at 1em so sizing remains predictable.

Color helper

Tokens such as .Primary-600, .icon--muted, or .icon--primary set currentColor so you never hardcode fills inside the SVG.

data-icon

The attribute name matches the registry entry; icon.js warns when it cannot find a template in icons.svg.

At runtime icon.js scans [data-icon], clones the template with cloneNode(true), clears the span, and appends the SVG. No <use> references survive in the page.

Usage examples

Basic inline icon

The default pattern for a standalone glyph.

<span class="icon icon-2xl Primary-600" data-icon="car"></span>

Button icon

Pair the glyph with visible text; the icon stays decorative with aria-hidden.

<button class="btn moi-btn--Primary d-inline-flex align-items-center gap-2">
  <span class="icon icon-2xl Primary-600 icon--start" data-icon="car" aria-hidden="true"></span>
  Drive
</button>

Label icon

Place the glyph before label text so the visual cue matches the accessible name.

<label class="d-flex align-items-center gap-2" for="destination">
  <span class="icon icon-2xl Primary-600 icon--start" data-icon="car" aria-hidden="true"></span>
  Destination
</label>

List item

List entries can adopt the helper classes for consistent spacing.

<li class="d-flex align-items-center gap-2">
  <span class="icon icon-2xl Primary-600 icon--start" data-icon="car" aria-hidden="true"></span>
  Vehicle status
</li>

Decorative icon

Keep the glyph untilted from screen readers when it is purely decorative.

<span class="icon icon-2xl Primary-600" data-icon="car" aria-hidden="true"></span>

Semantic color

Give the host an accessible label when the icon itself carries meaning.

<span role="img" aria-label="Vehicle" class="icon icon-2xl Primary-600" data-icon="car"></span>

Start/end alignment

.icon--start and .icon--end insert inline gap without extra markup. 

<div class="d-flex align-items-center gap-2">
  <span class="icon icon-2xl Primary-600 icon--start" data-icon="car" aria-hidden="true"></span>
  Inline label
  <span class="icon icon-2xl Primary-600 icon--end" data-icon="car" aria-hidden="true"></span>
</div>

Accessibility

Decorative icons

  • Add aria-hidden="true" when the glyph is purely decorative.
  • Keep the host element empty so the loader can inject the SVG.
  • Provide visible text elsewhere to carry the meaning when you hide the icon.

Meaningful icons

  • Use role="img" plus aria-label or pair the icon with nearby text.
  • Avoid duplicating the SVG; icon.js is the single source of the shape.
  • Combine color with text or badges so meaning survives even without tint.

DesignOps Governance — Iconography

Governance keeps every glyph aligned to the registry, loader, and SCSS helpers.

Best practices

  • Always add the glyph to icons.svg before referencing it with data-icon.
  • Relationships between icon, label, and container live in the UI; do not copy SVGs into components.
  • Keep names clear (e.g., action/search, status/success) and unique.

Do / Don't

  • Do rely on data-icon plus helper classes for color and size.
  • Do reuse tokens such as .Primary-600 or .icon--muted for states.
  • Don’t paste raw <svg> or use <img> for these glyphs.
  • Don’t hardcode fills or strokes inside the registry; let currentColor do the work.

Troubleshooting

  • Icon not showing? Confirm icons.svg is reachable and icon.js is loaded before the markup.
  • Console warns about missing icon name? Check for typos in data-icon and registry entries.
  • Size wrong? Apply a sizing helper such as .icon-2xl or remove conflicting CSS from the host.
  • New glyph still stale? Clear the cache so the loader fetches the updated registry.

Styling & theming

The helper classes come from icon.scss (compiled into main.css).

Sizing

  • .icon enforces inline-flex centering, line-height, and a 1.25rem default size.
  • Use the scale from .icon-sm through .icon-4xl to keep sizes predictable.
  • .icon--start/.icon--end add inline margins so glyphs align without extra wrappers.

Color & states

  • Semantic tokens (.Primary-600, .icon--muted, .icon--primary) drive currentColor.
  • Hover, focus, and disabled states come from the component classes that toggle these tokens; the SVG stays untouched.
  • The optional .icon--stroke helper keeps the fill unset and applies stroked lines via currentColor.

RTL notes

Glyph meanings stay constant while .icon--start and .icon--end handle inline spacing when the direction flips.

  • Do not move the icons themselves; use inline helpers for direction-aware alignment.
  • Icons keep the same color tokens in RTL and LTR; no token swap required.
  • Ensure icons inside buttons/links remain in the same reading order across directions.

Reference sources

  • Registry: icons.svg contains the inline SVG glyphs and their data-icon names.
  • Loader: icon.js fetches the registry, registers templates, and clones them wherever the DOM supplies data-icon.
  • Styles: css/main.css (compiled from icon.scss) defines .icon, size helpers, semantic colors, and spacing utilities.
