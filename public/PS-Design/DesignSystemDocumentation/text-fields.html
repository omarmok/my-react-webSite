Text Inputs & Selects

This page maps text inputs and dropdown selects using the same layout as buttons. Each field is built on the core .form-control and .form-select styles, layering icons and helper text where needed.

Overview

Inputs rely on the shipped Bootstrap styles .form-control and .form-select. Icons sit inside a .form-content wrapper and helper/error messaging uses .form-text or Bootstrap feedback classes.

Keep the 44px height, radius-sm , and 1px border defined in CSS.
State cues come from existing border/box-shadow rules (focus, disabled, error).

Usage rules

  • Keep the 44px height, radius-sm, and 1px border as defined in the base SCSS.
  • Start with .form-control/.form-select and layer icons inside .form-content.
  • Hook helper or error text via aria-describedby.
  • Apply .has-leading / .has-trailing to maintain icon spacing.

Do & Don't

Do

  • Reuse the existing SCSS mixins for every state.
  • Link helper/error text via aria-describedby when needed.
  • Treat icons as decorative (aria-hidden="true") and give close buttons a clear label.

Don't

  • Don’t alter height/borders inline; adjust via variables.
  • Avoid inventing new status colors.
  • Do not strip focus styles or keyboard support.

Anatomy

Each field is made of a label, the input container, optional leading/trailing icons, and helper or error text.

  • Label: Use <label for> that matches the field id.
  • Container: .form-control or .form-select for consistent height, padding, and border.
  • Leading icon: Placed inside .form-content with text indent.
  • Trailing icon: Action or state affordance (toggle visibility, dropdown chevron).
  • Helper/Error: .form-text or .invalid-feedback tied via aria-describedby.

Accessibility

Ensure labels and messaging are connected; interactive icons need aria-label.

  • Tie every label to a unique id.
  • Use aria-describedby for helper or error text.
  • Set aria-invalid="true" when showing error.
  • Icon buttons (toggle eye, chevron) need an aria-label.
  • Dropdown uses role="listbox" with role="option" items.

Edge cases

  • Monitor container height and line height for multi-line fields.
  • Toggle buttons must update their aria-label when the state changes.
  • For filters, pair .form-select with keyboard-friendly option lists.
  • RTL contexts require dir="rtl" and inline padding adjustments for icons.

Developer API

Define form fields, icons, and helper text using the classes listed below. 

<label for="username">اسم المستخدم</label>
<div class="input-wrapper has-leading has-trailing">
  <span class="input-icon leading" aria-hidden="true">...</span>
  <input class="form-control" id="username" type="text" aria-describedby="username-helper">
  <i class="fa fa-check input-icon trailing" aria-hidden="true"></i>
</div>
<p class="helper-text" id="username-helper">Helper text</p>
  • Each field starts with .form-control or .form-select and matching label.
  • Icons live inside .input-icon within .input-wrapper.
  • aria-describedby links helper/error text, and error messages use .invalid-feedback.
