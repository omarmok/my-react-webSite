Typography

Overview

Typography utilities are generated in _mixins.scss using size steps from xs to 7xl and weights 400-700. Use font-*-* to apply size and weight quickly, and the cairo class in _changeFontClasses.scss switches the UI to IBM Plex Sans Arabic.

Accessibility

  • Maintain text contrast that meets WCAG AA.
  • Use md or larger for long-form body text.
  • Avoid all-caps in Arabic and bilingual UI.
  • Keep line length around 45-80 characters for readability.

Usage rules & practical guidance

Apply these tokens and line heights to guarantee MOI readability while keeping bilingual surfaces consistent.

Usage rules

  • Map every surface-level heading to a semantic token before adjusting weight or size.
  • Use font-md-400 or larger for long-form paragraphs to preserve the 16px minimum.
  • Apply line-height-body whenever text is grouped with modular spacing tokens.
  • Keep font-*-* utilities in _mixins.scss synchronized with this guide.
  • Reuse the same token names across Arabic and English layouts to avoid fragmentation.

Do / Don't

  • Reference font-*-* utilities instead of inline font adjustments.
  • Keep uppercase limited to Latin tokens; avoid all-caps for Arabic strings.
  • Validate screen readers and contrast for each semantic level.
  • Don't create intermediate sizes outside this token set.
  • Don't override tokens in component themes without governance approval.
  • Don't shrink body text below font-md-400 or line-height 1.4.

Governance notes

  • Token updates need MOI DesignOps sign-off and documentation in this guide.
  • RTL review and Arabic QA are mandatory before release.
  • Keep _mixins.scss loops and component docs aligned with this page.

RTL notes

Ensure the font utilities behave the same across Arabic and English layouts, and keep the documented line heights intact.

  • Don’t flip token names; reuse the same font-*-* class in both RTL and LTR views.
  • Verify that RTL alignment and manual right-to-left rules don’t break letter spacing or line height.
